# Surrenderless — project state (source of truth)

This document aligns Cursor, ChatGPT, and humans on **product intent**, **current reality**, and **where to invest**. It describes the repo as of its last update; verify against code when in doubt.

---

## 1. True product goal

**Surrenderless is a chat-first action assistant.** The user should eventually **only chat** with Surrenderless — not manually hop across forms, tabs, and external sites as the primary experience. The user **chats**; the app:

- Gathers facts **conversationally**
- **Structures** the case
- **Collects and organizes** proof (screenshots, receipts, emails, and similar — tied to the case in-app)
- **Prepares** submissions, messages, and complaints for review (in-app previews and drafts today; more automation later where feasible)
- Gets **approval**
- **Submits or queues** actions when technically (and legally) possible
- **Tracks** confirmations, filings, and follow-ups
- **Carries** the issue from beginning to end

Existing **`/justice/*`** pages (intake form, evidence page, prep surfaces, etc.) are **interim scaffolding** toward that single conversational shell — not the final product shape.

Anything that permanently treats “open another tab and paste into a government form” as the final experience is **out of scope for the long-term product**, even if it is useful as an interim step.

### Known limitations (non-exhaustive)

- **Large saved-case lists:** `GET /api/justice/cases` paginates (**`limit`** / **`offset`**, **`has_more`**); **`/justice/cases`** offers **Load more cases**. Very large offsets or huge histories can still be slow — prefer reasonable page sizes.
- **Dual timeline:** Session + DB can drift if sync fails or multiple tabs are used.
- **Pre-server case id:** Intake can generate a non-UUID id in edge environments; evidence/task APIs require UUID `case_id`. **`/justice/chat-ai` update commit** only **PATCH**es the server when the active id is already a UUID; local-only ids still get **`STORAGE_INTAKE`** updated.
- **Structured form edit:** **`/justice/intake`** submit still always uses **create** commit — editing a loaded case via the form can still duplicate cases (chat-ai **update** path does not apply there yet).
- **Chat-ai evidence guidance:** **Link + copy only** on **`/justice/chat-ai`** — no **`GET /api/justice/evidence`** on that page; users save **`justice_case_evidence`** rows on **`/justice/evidence`** (requires signed-in + **UUID** **`case_id`**). **`/justice/evidence`** remains interim scaffolding until proof is fully conversational in-app.
- **Prepared next action on plan:** **`pickPreparedNextAction`** is **page-local**; primary CTA always routes to **`/justice/packet`** for in-app review; optional **`detailHref`** secondary links to merchant/prep pages. Does **not** persist a chosen step or read preview **`destination_label`**. Users still **manually** prep, copy, and file on external sites — **no** auto-submit, queue, filing workers, or implication that Surrenderless has filed/submitted (§2 **Plan — prepared next action**).
- **RLS vs Clerk:** Child-table RLS policies target Supabase Auth `uid`; app uses **Clerk id + service role** — fine for current server-only access, confusing if client-side Supabase is introduced without redesign.
- **Analytics / profile tables** not versioned in this repo’s migrations.

## 7. Development workflow

1. **Branch:** Do feature work on **`cursor-dev`** (or team-agreed equivalent), not directly on `main` for risky changes.
2. **Typecheck:** Run **`npx tsc --noEmit`** before commit/PR.
3. **Test:** Exercise flows locally (intake → preview → plan → cases → evidence/tasks/filings as relevant).
4. **Commit:** Clear, scoped commits.
5. **Push** `cursor-dev` and open a **PR to `main`**.
6. **Merge** after review.
7. **Sync:** Merge or rebase **`main` back into `cursor-dev`** so the long-lived branch stays current.

Adjust branch names if the team renames them; the pattern is **integration branch → PR → main → back-merge**.
